# 微信公众号写作工作区

本目录用于存放微信公众号文章的写作与准备材料。
采用“**一篇文章 = 一个独立章节包**”的方式组织：正文 Markdown、图片、草稿、资料、数据分析等都放在同一个文章目录下，方便迁移、复盘与长期维护。

## 目录结构

- `posts/YYYY/YYYY-MM-DD-<slug>/`
  - `YYYY-MM-DD-<slug>.md`：文章正文
  - 该文章相关的所有附件：图片、示意图、笔记、草稿、数据文件等（同级即可，不强制再分子目录）
- `docs/`：跨文章的公共文档（发布节奏分析、计划等）
- `data/`：导出数据 / 生成数据（不一定对应单篇文章）

## 命名规范

文章目录名与 Markdown 文件名必须同名：

- `YYYY-MM-DD-<kebab-case-english-slug>/YYYY-MM-DD-<kebab-case-english-slug>.md`

Slug 规则：

- 只用英文单词（不要拼音，不要中文）
- 尽量简短，能省略的虚词就省略，避免冗长
- 用 `-` 分隔，不要空格
- 常见技术缩写可用（如 `cli`、`ai`、`llm` 等）

## 写作流程（建议）

1. 在 `posts/` 下对应年份创建文章目录。
2. 在目录内创建同名 Markdown 文件作为正文。
3. 准备过程中的材料直接丢在该目录同级（图片、对比表、草稿、数据、引用截图等）。
4. 正文引用本地文件使用相对路径，比如 `./diagram.png`。
